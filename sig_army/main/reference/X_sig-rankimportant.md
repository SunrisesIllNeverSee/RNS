## SNR Comparison Models

### Message-Based SNR

```
Message-Based SNR = Signal messages / Total messages
```

**Strengths:**
- Quick structural overview of a session
- Marks pivot points, density bursts, drift events
- Ideal for class identification (e.g., "this thread ran at Transmitter-class density")

**Limitations:**
- Treats all messages equally (3 words or 3,000 tokens)
- Misses nuance when payload size varies

### All Four Models Compared

| Model | Basis | Strength | Use Case |
|-------|-------|----------|----------|
| **Message-Based SNR** | Signal messages / Total messages | Speed; clarity at thread-level | Thread audits, live sessions, class ID |
| **Token-Based SNR** | Signal tokens / Total tokens | Precision; accounts for payload size | Document audits, vault integrity, leaderboard scoring |
| **Word-Based SNR** | Signal words / Total words | Blunt, language-redundant | Rarely used (fallback for non-token systems) |
| **Prompt Compression Ratio** | Compressed meaning / Prompt length | True compression benchmark | Transmitter-class testing, leaderboard proof |

### Analogy
- Message-SNR = looking at frames in a film reel
- Token-SNR = measuring the ink on the reel itself
- Prompt compression = asking: "Did we condense the entire film into one perfect scene?"

The patent defines a Signal Economy Portal — a "transactional layer for token rewards, licensing, and value exchange within the system." Metrics like resonance and TPW can function as "tradable, licensable economic units."

### Proving the Patent
The patent is profound and abstract — strength (hard to copy) and weakness (hard to prove without prototype). Bridge the gap by showing verifiable data in a live environment. Build the simplest possible leaderboard, show verifiable data to potential partners.
